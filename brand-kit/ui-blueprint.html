01 — Foundations

Identity

SetList reads premium, considered, quietly confident — closer to Whoop or Strava's editorial moments than gym-app maximalism. Restrained color, real typographic hierarchy, a light white ground. No gradients, no drop shadows, no glow — anywhere.

Wordmark
SetList

Set in near-black, List in brand blue — Big Shoulders Display 800, no gap between the words. Matches the in-app header exactly (Set<em>List</em>). Shipped as logo.svg with the font embedded.

App icon
SL
SL
SL

Two-tone SL monogram — ink S + blue L, same Big Shoulders 800, on a plain white field. Ships as icon-192/512, apple-touch-icon, favicon.svg. (The old purple-gradient bolt is retired — wrong color family, uses a gradient the brand avoids.)

05 — Design canvas

Screens that don't exist yet

The gaps worth designing next — drawn from the deferred Phase 2 items, the roadmap's differentiator strategy, and holes in the current flow. Each notes why it matters and where it maps.

Lean into the moat. Every exercise is permanently tagged with creator_handle + creator_platform — that's SetList's structural edge over 10+ extraction competitors. The creator & type browse screens below turn that data into a real differentiator, not metadata.

Creator page

All workouts & exercises from one @handle, in one place — the "your handle is already a category" pitch made real. Follow, browse, attribute.

MAPS TO · Layer 2 differentiator · Phase 5 community

Browse by type

Filter the whole library by the closed exercise-type set — strength / cardio / core / mobility / plyometric. Discovery beyond a single workout.

MAPS TO · Layer 2 · creator/type browsing

Profile & settings

Today "account" is a lone sign-out row on Progress. A real profile: display name, email, notifications, sign out, delete account.

MAPS TO · deferred Phase 2 · "dedicated profile screen"

Per-exercise progress

Tap an exercise → a time-series of weight/volume for it. The current Progress screen shows PRs + weekly volume, but not the trend per movement.

MAPS TO · deferred Phase 2 · "progress-over-time charts"

Resume in-progress

A refresh mid-workout currently loses the session. A "continue your workout?" banner/prompt to restore it.

MAPS TO · deferred Phase 2 · "resumable workout"

Offline & error states

No-signal library (gym reality), a failed-load retry, a cloud-sync-failed banner. Designed states, not raw toasts.

MAPS TO · deferred Phase 2 · "offline read cache"

Upgrade / paywall

Free (~5 extractions/mo) vs Pro (~$4.99/mo). The reel-to-card moment as the hero of the pitch.

MAPS TO · Phase 4 · Stripe paywall

Branded auth email

Magic-link emails still ship from the default "Supabase Auth" template. Design the SetList sign-in email before beta testers see it.
